INSTALLING DBT LOCALLY

How to install dbt locally on Windows

The way that data models are transformed has been revolutionized over the past few years and a tool that has been at the forefront of this change is dbt (data build tool). There are currently a handful of guides on how to install this tool, however most of them lack the necessary depth to cover all aspects of the process, making it hard for people who are new in data to follow.

This article aims to walk you through each single step of successfully installing this tool in your virtual environment and will build upon dbt core's installation guide. There are multiple adapters available depending on the data warehouse that you use, however we will leverage PostgreSQL, a free and open source relational database management system (RDBMS), which you will install in your local machine. Therefore, you can create and work on your own small database without the need to create any cloud account. This could be especially useful for people who just want to get hands-on experience with dbt and are in their first steps of their data journey or for those who want to further practice their data modelling skills.

A few callouts:

  1. This example was done on a Windows machine.
  2. This approach is tailored to people new in dbt and does not follow the Docker installation process but rather pip.
  3. At the time of writing, the latest dbt version is 1.3.
  4. This example was carried out using the CLI and not dbt cloud.
  5. Prerequisite: A code editor. I personally use VS Code.

Install Postgres

In this example we will download and use postgres locally, one of the most widely used RDBMS. Firstly, download postgres. Choose your operating system and once the download is complete proceed by installing it. Create your credentials and I would also strongly recommend downloading a GUI such as DBeaver to execute your queries.

Install Python and PIP

The next step is to install Python and PIP. Before proceeding, check which are dbt-core's current compatible Python versions on dbt's website. Then head to https://www.python.org/downloads/ and download the latest compatible Python version. Once the installation starts, make sure to select add python to PATH as well as to environment variables.

Install dbt

The last thing to be installed is dbt and specifically the postgres adapter. Execute the following command in your terminal:

$ pip install dbt-postgres

Installing dbt will take a couple of minutes. Once the installation is complete, make sure that it has been installed correctly by running the following command, which will print out the dbt version in your local machine.

$ dbt --version

Configuring dbt

A new folder, .dbt, should have been created outside of the dbt project containing a profiles.yml file. Dbt uses profiles to connect to the database, therefore it is of the utmost importance to configure it with the correct credentials. Open the file and update each respective field and use the credentials used when setting up PostgreSQL.

To ensure that a connection can be established, execute the following command in the terminal:
$ dbt debug

If the configuration was set up correctly, executing the dbt debug command should return something like this: With dbt set up, you can start creating your own data models! In case you are looking for datasets to download and practice both your PostgreSQL and data modelling skills, I would recommend downloading the jaffle shop repository, which contains raw data that you can use to get started with dbt!

Final Words

This article shows how to install and configure dbt locally, by installing the postgres adapter via pip. Happy data modelling!
